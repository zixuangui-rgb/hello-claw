# 让龙虾自己落脚本和设置定时任务：可直接复制的提示词

这份文件不是代码，而是给 OpenClaw / 龙虾的操作提示词模板。

它解决两个问题：

1. 让龙虾把你上传的脚本文件保存到当前工作目录
2. 让龙虾自己创建每天早上 9 点执行的邮件摘要定时任务

---

## 用法 1：先让龙虾自己保存脚本和环境变量模板

把 `yesterday_mail_report.py` 作为附件上传给龙虾，然后发送下面这段提示词：

```text
我刚刚上传了一个脚本文件，请你帮我把它接入到当前工作目录。

要求你完成下面这些事：

1. 在当前工作目录创建 `scripts/` 目录（如果已经存在就复用）
2. 把我上传的 `yesterday_mail_report.py` 保存到 `scripts/yesterday_mail_report.py`
3. 在当前工作目录创建 `.env.example`，写入这些占位符：
   MAIL_HOST=imap.163.com
   MAIL_PORT=993
   MAIL_USER=your_name@163.com
   MAIL_PASSWORD=replace_with_authorization_code
   MAIL_FOLDER=INBOX
4. 不要把真实授权码写进文件
5. 完成后告诉我：
   - 脚本实际保存到了哪里
   - `.env.example` 保存到了哪里
   - 我下一步应该复制什么文件并填什么变量

如果当前工作目录里已经有同名文件，先告诉我差异，再决定是否覆盖。
```

---

## 用法 2：再让龙虾自己创建每天早上 9 点的 cron

等脚本已经放到当前工作目录之后，再发送下面这段提示词：

```text
请帮我在当前 OpenClaw 环境中创建一个每天早上 9 点执行的定时任务。

任务目标：
- 运行当前工作目录下的 `scripts/yesterday_mail_report.py`
- 读取脚本输出的 JSON
- 只汇报昨天 00:00-23:59 的邮件

输出要求：
1. 按四类整理：
   - 需要我处理
   - 安全提醒
   - 系统通知
   - 可忽略邮件
2. 每封邮件只保留：
   - 主题
   - 发件人
   - 时间
   - 一句话摘要
3. 最后增加一个「今天需要优先处理的事项」小节
4. 如果 `total_messages = 0`，直接告诉我昨天没有需要汇报的新邮件

执行要求：
- 用 cron 方式创建
- 触发时间设为每天早上 9 点
- 使用隔离会话
- 创建后把 cron 任务名称、cron 表达式和查看任务的命令一起告诉我
```

## 用法 3：一步到位，让龙虾同时做两件事

如果你想一次性交给龙虾处理，也可以把脚本附件上传后直接发这段：

```text
我上传了一个 `yesterday_mail_report.py` 脚本文件。请你在当前工作目录里完成一整套接入：

1. 创建 `scripts/` 目录并保存脚本到 `scripts/yesterday_mail_report.py`
2. 创建 `.env.example`，写入邮箱连接变量占位符，但不要写入真实授权码
3. 告诉我应该如何复制 `.env.example` 成 `.env` 并填入我的真实邮箱变量
4. 在确认脚本路径正确之后，创建一个每天早上 9 点执行的 cron 任务
5. 任务执行时：
   - 运行 `scripts/yesterday_mail_report.py`
   - 读取脚本输出的 JSON
   - 只汇报昨天邮件
   - 按“需要我处理 / 安全提醒 / 系统通知 / 可忽略邮件”四类整理
   - 最后给出「今天需要优先处理的事项」
   - 如果没有邮件，就直接汇报昨天没有需要关注的新邮件

完成后请用清单告诉我：
- 脚本存放路径
- `.env.example` 路径
- cron 任务名称
- cron 表达式
- 我后续查看、修改、删除任务分别用什么命令
```

---

## 什么时候优先用这种提示词方式

这种方式适合：

- 你想让龙虾自己完成文件落盘
- 你不想手动敲 cron 命令
- 你更希望以“交互任务”而不是“手工部署”的方式推进

如果你更在意可控性，还是可以直接用命令方式。  
这两种方式不冲突，命令式是底层兜底，提示词式是更省手的操作入口。
