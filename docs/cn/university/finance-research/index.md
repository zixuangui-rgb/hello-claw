# 更多场景实战：金融数据分析助手

> **适用场景**：你每天要看行情、读研报、跟踪新闻，但信息量太大，容易“看了很多却没有决策输入”。本文对应 README 场景“金融数据分析”。

这不是投资建议，而是帮助你搭好一条“采集 → 筛选 → 总结 → 预警”的研究流水线，让每条结论都有来源与时间戳。

## 1. 这篇能帮你什么

- 自动收集行情、新闻与公告并归类成可比较的主题
- 输出带来源、时间的当日摘要，方便复盘
- 异常波动能自动推送并标明“触发条件 + 关联标的”
- 可以把日报沉淀到飞书，方便每天回顾

## 2. 先复制这一句给龙虾

```text
请帮我搭一个“金融数据分析助手”：
1) 跟踪的标的池是：________（示例：NVDA, TSLA, 3 家新能源）
2) 每天收集行情 + 新闻 + 重大公告，输出一条结构化日报：市场概览、重点标的、关键事件、风险提示、次日关注
3) 当价格波动、成交量或公告关键词超过阈值时立即推送预警；结论必须附带来源链接+采集时间
4) 所有输出都用中文，最后补一句“仅用于研究辅助，不构成投资建议”
```

## 4. 跑通后你会看到什么

```text
【市场概览】
纳指收涨 1.8%，半导体板块量能放大，数据已归入“芯片+AI”主题

【重点标的】
NVDA +4.2%（量能放大，AI 推理芯片逻辑）；TSLA -2.4%（监管关键词触发预警）

【风险提示】
TSLA 监管消息仍在发酵，建议重点观察成交量与公告关键词

【次日关注】
继续监控 NVDA 推介会进度 + 关注是否出现新的监管关键词
```

以上内容都带来源链接、采集时间，方便你末尾直接贴到研究日志。

## 5. 怎么一步步配出来

1. 先选 10 到 20 个核心标的/主题，给每个设定涨跌与成交量阈值
2. 设定日报模板（市场概览、重点标的、关键事件、风险提示、次日关注）
3. 把 `tavily-search` 用在新闻与公告检索，`summarize` 用在研报压缩
4. 设置异常检测规则（如日内波动超过 5%、成交量放大 2 倍，或出现禁用词）
5. 每条结论附带来源链接 + 采集时间，并通过 `feishu-send-message` 推送当天摘要或预警

## 6. 如果没有现成 Skill，就让 Claw 帮你造

如果你找不到稳定的行情 skill，先别慌，直接把这句话发给龙虾：

```text
请帮我生成一个最小可用的行情抓取 skill，第一版只做 3 件事：拉取核心标的价格、判断是否触发阈值、输出带来源和时间的日报草稿。
```

等它先把第一版思路起好，再考虑落成下面这个骨架：

```text
market-watch/
├── SKILL.md
└── scripts/
    └── quote.py
```

`SKILL.md` 说明该技能负责拉行情、做阈值判断、生成摘要；`scripts/quote.py` 里第一版只做三件事：1) 拉取核心标的价格；2) 判断是否触发阈值；3) 输出包含来源与时间的日报草稿。

## 7. 再往下优化

- 把多个数据源的结果做去重/合并，避免同一个事件被重复推送
- 加入“主题总览”视图，把相关标的打包输出，支持按行业/策略拆分
- 让预警带上“操作建议”栏，例如“需要人工复盘 / 继续观察 / 加仓警告”

## 8. 常见问题

**Q1：数据源之间数值不一致怎么办？**
A：先定义“主数据源”，其他仅做交叉验证；摘要里标注“数据来源”字段。

**Q2：预警太多，读不过来怎么办？**
A：增加冷却时间、按照优先级分级推送，或只在异常分数超过 80 分才告警。

**Q3：总结内容太泛，难以行动怎么办？**
A：在模板里强制加入“次日关注点” + “触发条件”，结论才有行动价值。
