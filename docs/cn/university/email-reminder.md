# 🧪 龙虾大学：邮件提醒实战（高优先级邮件不再漏）

> **适用场景**：每天收几十封邮件，重要邮件总被淹没在信息流中；老板临时找你、客户紧急签约、合作伙伴催回复，总是后知后觉。

这篇文章不是简单的"邮件分类"教程，而是一套**生产级别的邮件处理工作流**。它融合了 OpenClaw 的三大核心能力：**实时监控**（AgentMail）、**智能分级**（LLM 推理）、**自动闭环**（Todoist + 消息推送），让你的邮箱从"信息黑洞"变成"效率引擎"。

---

## 1. 你将得到什么（真实场景价值）

跑通后，你会拥有一个**7×24 小时在线的邮件助理**：

### 场景 1：老板深夜发邮件，第二天早上你已处理完
- **问题**：老板凌晨 2 点发邮件，你第二天早上看到时已经过去 8 小时
- **解决**：邮件到达瞬间，通过飞书/Telegram 推送到手机，附带"建议响应时间"（如"需 2 小时内回复"）

### 场景 2：客户合同邮件被误判为垃圾邮件
- **问题**：客户发来合同草案，被邮箱过滤器误判为广告，直到截止日期前 1 小时才看到
- **解决**：P0 级别邮件（含"合同"、"协议"、"签署"等关键词）强制推送到你的优先级通道

### 场景 3：每周五下午 5 点，自动生成"本周未处理邮件清单"
- **问题**：周末前总有一些邮件被遗忘，周一早上手忙脚乱
- **解决**：定时任务每周五 17:00 自动扫描 P1/P2 级别未处理邮件，生成待办清单

### 场景 4：群发邮件不 spam，但重要信息不遗漏
- **问题**：100 人参会的会议通知，你不想每封都点开，但怕错过关键信息
- **解决**：群发邮件自动摘要，只提取"时间、地点、行动项"三要素

## 2. 技能选型：为什么这些是"最小可行集"？

```bash
clawhub install skill-vetter      # 安全守卫（必须第一个安装）
clawhub install agentmail         # 邮件监控（核心）
clawhub install todoist           # 待办管理（核心）
clawhub install gog               # Google 全家桶（可选，用于日历同步）
clawhub install tavily-search     # 联网搜索（可选，用于查询发件人背景）
```

### 为什么是这 5 个？

| 技能 | 不可替代性 | 替代方案风险 |
|------|-----------|-------------|
| **agentmail** | 唯一支持 IMAP 实时监控的技能 | 其他邮件技能需要手动轮询，无法实时 |
| **todoist** | 支持优先级、截止时间、标签的完整任务系统 | Notion 需要手动创建数据库，Obsidian 缺少提醒功能 |
| **skill-vetter** | 自动扫描技能是否窃取 API Key | 不安装可能被恶意技能盗号 |
| **gog** | Gmail + Calendar 无缝集成 | 单独用 agentmail 需要手动同步日历 |
| **tavily-search** | 快速查询发件人是否可信 | 不查询可能被钓鱼邮件欺骗 |

> **重要提示**：不要贪多！这 5 个技能已经足够支撑一套生产级工作流。装太多技能会导致：
> - 上下文膨胀，LLM 判断准确率下降
> - 响应速度变慢（每多 1 个技能，平均延迟增加 200-500ms）
> - 维护成本上升（每个技能都要关注版本更新）

## 3. 配置指南：从安装到生效的完整流程

### 3.1 账户连接（关键步骤）

#### AgentMail 配置（以 Gmail 为例）

```bash
clawhub install agentmail
```

安装后，OpenClaw 会提示你输入 IMAP 配置：

```
IMAP Server: imap.gmail.com
Port: 993
Username: your-email@gmail.com
Password: [应用专用密码，非登录密码]
```

> **为什么用应用专用密码？** Gmail 要求 2FA（双因素认证）开启后，必须使用应用专用密码。生成路径：Google Account → Security → 2-Step Verification → App passwords

#### Todoist 配置

```bash
clawhub install todoist
```

在 [Todoist Apps](https://todoist.com/prefs/apps) 生成 API Token，复制到 OpenClaw 提示符：

```
Enter Todoist API Token: [粘贴]
```

#### 验证连接状态

```bash
openclaw whoami
clawhub list
```

确保 `agentmail` 和 `todoist` 状态为 `active`。

### 3.2 邮件分级规则（核心逻辑）

在 `openclaw.json` 中添加自定义规则：

```json
{
  "skills": {
    "agentmail": {
      "rules": [
        {
          "name": "P0-紧急",
          "conditions": {
            "from": ["boss@company.com", "ceo@client.com"],
            "subject_contains": ["紧急", "今天", "截止", "合同", "付款", "签约"]
          },
          "actions": {
            "priority": "high",
            "todo_due_in_hours": 2,
            "notify": true,
            "notify_channel": "feishu"
          }
        },
        {
          "name": "P1-重要",
          "conditions": {
            "from": ["customer@*.com", "partner@*.com"],
            "subject_contains": ["项目", "合作", "需求", "方案", "报价"]
          },
          "actions": {
            "priority": "medium",
            "todo_due_in_hours": 8,
            "notify": true,
            "notify_channel": "telegram"
          }
        },
        {
          "name": "P2-通知",
          "conditions": {
            "subject_contains": ["周报", "月报", "会议纪要", "通知"]
          },
          "actions": {
            "priority": "low",
            "todo_due_in_hours": 24,
            "notify": false,
            "summarize": true
          }
        }
      ]
    }
  }
}
```

> **规则设计原则**：
> 1. **从具体到抽象**：先写死几个关键联系人（如老板、核心客户），再用通配符（`*.com`）覆盖更多场景
> 2. **渐进式优化**：第一周用严格规则（只对明确关键词触发），观察漏判邮件后逐步放宽
> 3. **避免冲突**：规则按顺序匹配，先匹配到的规则生效。P0 规则必须放在 P1 之前

## 4. 第一次跑通：从手动验证到自动化

### 4.1 手动验证（关键！）

**不要直接启用自动规则**！先手动验证逻辑正确性：

```
请检查我最近 2 小时的新邮件，并执行：
1) 按 P0/P1/P2 分级（规则见 openclaw.json）
2) P0 邮件输出：发件人、主题、一句话摘要、建议处理时限
3) 为 P0/P1 邮件创建待办（截止时间 = 当前时间 + todo_due_in_hours）
4) P2 邮件只输出摘要
5) 输出最终清单：P0 邮件数、P1 邮件数、P2 邮件数、已创建待办数
```

观察输出是否符合预期，特别是：
- 关键联系人是否被正确识别
- 关键词匹配是否准确
- 待办创建是否成功

### 4.2 调试技巧

#### 技巧 1：查看邮件原始内容

```
请展示最近一封邮件的完整原始内容（包括所有 header）
```

这能帮你诊断：
- 发件人地址是否被伪装（检查 `From` header）
- 邮件是否被转发（检查 `Received` header）
- 是否包含隐藏的附件链接

#### 技巧 2：测试规则匹配

```
请模拟检查以下邮件，判断应匹配哪条规则：
发件人：boss@company.com
主题：紧急：合同签署截止时间提前到今天 18:00
```

如果判断错误，调整规则中的 `from` 或 `subject_contains`。

### 4.3 启用自动监控

验证通过后，启用自动监控：

```
请启用邮件自动监控模式：
- 每 30 秒检查一次新邮件
- 匹配 P0/P1 规则的邮件立即推送到飞书
- 匹配 P2 规则的邮件每小时汇总一次
- 创建待办后回复"已处理：[主题]"
```

## 5. 高级场景：从"能用"到"好用"

### 场景 1：防止"消息轰炸"

**问题**：P0 规则太宽泛，导致每封邮件都推送，最终你忽略所有通知。

**解决方案**：增加"双重触发"条件：

```json
{
  "name": "P0-紧急（双重触发）",
  "conditions": {
    "from": ["boss@company.com"],
    "subject_contains": ["紧急", "今天", "截止"],
    "body_contains": ["请立即", "马上", " urgent", "ASAP"]
  }
}
```

**原理**：即使发件人是老板，如果邮件内容没有紧急关键词，也不触发 P0。

### 场景 2：去重防重复

**问题**：同一封邮件被多次推送（比如你已读但未处理，下次检查时又触发）。

**解决方案**：在 `agentmail` 规则中添加 `dedup_key`：

```json
{
  "name": "P0-紧急",
  "conditions": { ... },
  "actions": {
    "dedup_key": "{{ from }}-{{ subject }}-{{ date | date('YYYY-MM-DD') }}",
    "priority": "high",
    "todo_due_in_hours": 2
  }
}
```

**原理**：`dedup_key` 会计算邮件的唯一标识，相同标识的邮件不会重复创建待办。

### 场景 3：智能摘要生成

**问题**：P2 邮件（如会议通知）需要人工点开才能提取关键信息。

**解决方案**：使用 `summarize` 技能 + `agentmail` 自动触发：

```
请为 P2 邮件自动生成摘要，格式：
【会议主题】xxx
【时间】xxx
【地点/链接】xxx
【行动项】xxx（如有）
```

### 场景 4：定时汇总（下班前未处理清单）

在 `openclaw.json` 中添加定时任务：

```json
{
  "cron": {
    "jobs": [
      {
        "name": "下班前未处理邮件清单",
        "schedule": "0 17 * * 1-5",
        "prompt": "请检查所有待办事项中状态为 'pending' 且截止时间为今天的邮件，按 P0/P1/P2 分组输出清单，格式：
### P0（需 2 小时内处理）
- [ ] 邮件主题（发件人）

### P1（需 8 小时内处理）
- [ ] 邮件主题（发件人）

### P2（今日内处理）
- [ ] 邮件主题（发件人）

请在 17:30 前发送到飞书工作群。"
      }
    ]
  }
}
```

## 6. 常见问题与排障

### 问题 1：邮件没推送，但手动检查能发现

**诊断步骤**：

1. 查看 `agentmail` 日志：
   ```bash
   openclaw logs --follow
   ```
   关注 `agentmail` 的输出，是否有 `Connected`、`Fetching` 等关键词。

2. 检查 IMAP 连接：
   ```
   请检查 IMAP 连接状态
   ```
   正常输出应包含 `Status: connected`。

3. 手动触发检查：
   ```
   请立即检查新邮件
   ```

**常见原因**：
- IMAP 服务器配置错误（端口、SSL）
- 应用专用密码过期（Gmail 每 3 个月需重新生成）
- 邮件被邮箱服务器缓存（等待 5-10 分钟）

### 问题 2：待办创建失败

**诊断步骤**：

1. 检查 Todoist API Token：
   ```bash
   clawhub list | grep todoist
   ```
   确保状态为 `active`。

2. 手动创建待办测试：
   ```
   请为我创建一条待办：测试待办功能，截止时间今天 23:59
   ```
   如果失败，说明 Todoist 技能配置有问题。

**常见原因**：
- API Token 权限不足（需要 `data:write` 权限）
- Todoist 账户余额不足（免费版有待办数量限制）

### 问题 3：误判率高

**解决方案**：启用"人工确认"模式

```
请启用人工确认模式：
- P0 邮件先不推送，而是输出"建议推送：[主题]，原因：[关键词匹配]
- 你回复'是'或'否'决定是否推送
- 连续 5 次正确判断后，自动切换到全自动模式
```

**原理**：通过少量人工干预，让 LLM 学习你的判断标准，逐步提高准确率。

## 7. 进阶玩法：构建你的邮件处理系统

### 玩法 1：邮件响应效率周报

每周五自动执行：

```
请生成本周邮件响应效率周报：
1. 收到邮件总数、P0/P1/P2 分布
2. 平均响应时间（对比截止时间）
3. 未按时处理的邮件清单
4. 改进建议（如"周三下午响应时间最长，建议设置下午 4 点后自动提醒"）
```

### 玩法 2：发件人信用评分

```
请为我创建发件人信用评分系统：
- 首次发件人：默认 50 分
- 24 小时内回复：+10 分
- 关键词匹配准确：+5 分
- 误判（P2 邮件被推送到 P0）：-20 分
- 每周更新评分，低分（<30）发件人邮件自动降级
```

### 玩法 3：多邮箱统一管理

如果你有多个邮箱（工作、个人、订阅），可以配置多个 `agentmail` 实例：

```json
{
  "skills": {
    "agentmail-work": {
      "imap_server": "imap.company.com",
      "username": "your-name@company.com",
      "rules": [ ... ]
    },
    "agentmail-personal": {
      "imap_server": "imap.gmail.com",
      "username": "your-name@gmail.com",
      "rules": [ ... ]
    }
  }
}
```

## 8. 性能优化建议

### 优化 1：降低检查频率（节省 API 调用）

```
请将检查频率从 30 秒调整为 2 分钟（适合低优先级邮箱）
```

**原理**：每降低 1 次检查/分钟，月度 API 调用减少约 21,600 次。

### 优化 2：启用"仅新邮件"模式

```
请只检查自上次检查以来的新邮件（避免重复处理已读邮件）
```

**原理**：避免重复处理已读邮件，减少待办创建次数。

### 优化 3：限制邮件历史深度

```
请只检查最近 7 天的邮件（避免加载大量历史邮件）
```

**原理**：每减少 1 天历史深度，内存占用减少约 15MB。

这套工作流的核心价值不是"自动处理邮件"，而是**建立一种新的工作习惯**：

- **早上 9 点**：查看飞书推送的 P0 邮件，快速处理

- **每月 1 号**：清理低分发件人

**记住**：龙虾不是替代你工作，而是帮你**把时间花在最重要的事情上**。
